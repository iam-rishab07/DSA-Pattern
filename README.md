# 🚀 DSA Problem Solving Patterns

This repository contains my structured journey of mastering Data Structures & Algorithms (DSA) through core problem-solving patterns. Instead of memorizing 1000+ problems, I focus on the underlying strategies that solve entire classes of questions.

## 🧠 Patterns Covered


| Pattern | Description | Folder Link |
| :--- | :--- | :--- |
| **Two Pointers** | Opposite ends or different speeds (e.g., Target Sum, Valid Palindrome) | [/Two-Pointers](./Two-Pointers) |
| **Sliding Window** | Subarray/substring problems with a fixed or variable size | [/Sliding-Window](./Sliding-Window) |
| **Fast & Slow Pointers** | Cycle detection in Linked Lists or arrays | [/Fast-Slow-Pointers](./Fast-Slow-Pointers) |
| **Merge Intervals** | Dealing with overlapping time/range intervals | [/Merge-Intervals](./Merge-Intervals) |
| **Modified Binary Search** | Searching in sorted, rotated, or nearly-sorted arrays | [/Binary-Search](./Binary-Search) |
| **Tree BFS/DFS** | Level-order and depth-first traversals | [/Trees](./Trees) |
| **Top 'K' Elements** | Using Heaps to find the largest/smallest elements | [/Heaps](./Heaps) |

## 📁 Repository Structure
- Each folder is named after a **Pattern**.
- Inside each folder, you'll find `.cpp` (or your preferred language) files named after the problem.
- Most files include the problem statement and time/space complexity analysis in comments.

## 🛠 How to Use
1. Pick a pattern from the table above.
2. Read the implementation in the corresponding folder.
3. Try to solve similar problems on [LeetCode](https://leetcode.com) or [GeeksForGeeks](https://www.geeksforgeeks.org).
